- # Ethernet(イーサネット)
    - 有線LANの事実上の標準
    - 広範囲に使われた最初のLAN
    - 歴史
      - 1970年代
        - Xeroxによる原型の開発
          - 2.94Mbps
      - 1980年代
        - DIX Ethernet
          - 10Mbps
      - 1983年
        - IEEE802
          - **CSMA/CD**として規格化
          - 10Mbps
      - 2012年
        - IEEE802.3のタイトルが**Ethernet**になる
    - EthernetはCSMA/CDを使っている
      - Carrier Sense
        - 送信する前に確認する
          - チャンネルが使用可:送信を開始
          - チャンネルが使用中:使用可になるまで待つ
      - Collision Detection
        - 送信中も受信を続ける
          - 衝突を非検知:送信成功
          - 衝突を検知:送信中止・妨害信号を送出
      - Random Aaccess
        - 指数的バックおふ
          - 衝突検知後:ランダムな時間待ってから再送信
          - n回目の衝突:$k \epsilon \lbrace0,1, ・・・,2^{n-1}\rbrace$をランダムで選びk×(512bit分の時間)だけ待ってから再送信する
    - ケーブル長の制限
        ```
            |-----|            |-----|
            |pc A |            |pc B |
            |-----|            |-----|
              ||                 ||
              ||    遅延時間d      ||
               ===================
        ```
        - 遅延時間はケーブルの長さ二位依存する
          - 待ってる間、ずっと傍受し続ける必要がある
        - Ethernetの制限が生じる
          - ケーブル長は2500m
          - パケット長は最小512bit(64バイト)
    - 伝播遅延とケーブル長
        伝送速度を10Mbpsとするときに、イーサーネットにおける最小フレーム(64バイト)を衝突検出するためのケーブル長（短縮率0.77）の最大長$L_{max}$を求めよ

        $$ケーブル上の信号の速度\upsilon_s$$
        $$=3.0\times10^8 \lbrack ^m/_s \rbrack \times 0.77 = 2.3 \times 10^8 \lbrack ^m/_s \rbrack $$

        $\frac{64\times8}{10\times10^6}=5.12\times10^{-5}s$
        $0.77\times3.0\times10^8=2.3\times10^8m/s$
        $L_{max}=\frac{1}{2} \times5.12\times10^{-5}\times2.3\times10^8=5.9\times10^3m$
    - 伝播遅延とケーブル長
         伝送速度を10Mbpsとし、伝送遅延μsのリピータ2台で3本のケーブルを接続するとき,イーサーネットにおける最小フレームを衝突検出するためのケーブル（短縮率0.77）の最大長$L_{max}$を求めよ
        $
        t_s \geq \frac{6L}{v_s} + 4d_r
        $

        $
        L \leq \frac{1}{6}(t_s - 4d_r)v_s
        $

        $
        t_s = \frac{512\ \mathrm{bit}}{10^7\ \mathrm{bps}}
        = 51.2 \times 10^{-6}\ \mathrm{s}
        $

        $
        v_s = 3.0 \times 10^8 \times 0.77\ \mathrm{m/s}
        = 2.3 \times 10^8\ \mathrm{m/s}
        $

        $
        d_r = 9.0 \times 10^{-6}\ \mathrm{s}
        $

        $
        L_{\max}
        = \frac{1}{6}(51.2 - 4 \times 9) \times 10^{-6}
        \times 2.3 \times 10^8
        = 580\ \mathrm{m}
        $
   - Ethernetのフレーム構造
        ```
                              ________________MACアドレス_____________
                             ||                                     ||
                 7       1   ||       6                   6         ||  2        0-1500              0-46      4
             _____________________________________________________________________________________________________________
            |          |     |                   |                   |     |                     |          |             |
            | Preamble |     | Destination       |  Source           |     |       D a t a       |   Pad    |  CRC        |
            |          |     | address           |  address          |     |                     |          |             |
            |__________|_____|___________________|___________________|_____|_____________________|__________|_____________|

                          ^                                             ^
                          Stat of framae                          Type/Length of data field
        ```
        - 信号のタイミングを合わせるためにPreambleをフレームの前に送信する
            7バイトの**10101010**の後に1バイトの**10101011**を送出 
        - エラー検出にはCRCが使われる
          - エラーを検出したらフレームを破棄する
        - 宛先と送信元はそれぞれ6ビットの**MACアドレス**で指定する
          - 受信したフレームの宛先が**自分のアドレス**または**ブロードキャスト**なら**上位プロトコル**へ渡す
          - そうでなければ破棄する
        - フレーム長の最大は1500バイト
          - 最小は64バイトになるようにPadが付加される(64-(6×2+2+4)=46)
        - Type部分で**上位プロトコル**が指定される
    - 信頼性の低いコネクションレスのサービス
      - コネクションレス
        - 送受信ノード間でハンドシェークをしない
      - 信頼性が低い
        - 受信のーどは送信のーどに確認応答も否定応答も返さない
        - 上位層に渡すパケットには一部欠損があるかもしれない
        - 欠損はトランスポート層で対処する
