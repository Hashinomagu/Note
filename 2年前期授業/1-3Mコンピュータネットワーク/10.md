- # Ethernet(イーサネット)
    - 有線LANの事実上の標準
    - 広範囲に使われた最初のLAN
    - ### 歴史
      - 1970年代
        - Xeroxによる原型の開発
          - 2.94Mbps
      - 1980年代
        - DIX Ethernet
          - 10Mbps
      - 1983年
        - IEEE802
          - **CSMA/CD**として規格化
          - 10Mbps
      - 2012年
        - IEEE802.3のタイトルが**Ethernet**になる
    - ## EthernetはCSMA/CDを使っている
      - Carrier Sense
        - 送信する前に確認する
          - チャンネルが使用可:送信を開始
          - チャンネルが使用中:使用可になるまで待つ
      - Collision Detection
        - 送信中も受信を続ける
          - 衝突を非検知:送信成功
          - 衝突を検知:送信中止・妨害信号を送出
      - Random Aaccess
        - 指数的バックおふ
          - 衝突検知後:ランダムな時間待ってから再送信
          - n回目の衝突:$k \epsilon \lbrace0,1, ・・・,2^{n-1}\rbrace$をランダムで選びk×(512bit分の時間)だけ待ってから再送信する
    - ## ケーブル長の制限
        ```
            |-----|            |-----|
            |pc A |            |pc B |
            |-----|            |-----|
              ||                 ||
              ||    遅延時間d      ||
               ===================
        ```
        - 遅延時間はケーブルの長さ二位依存する
          - 待ってる間、ずっと傍受し続ける必要がある
        - Ethernetの制限が生じる
          - ケーブル長は2500m
          - パケット長は最小512bit(64バイト)
    - ## 伝播遅延とケーブル長
        伝送速度を10Mbpsとするときに、イーサーネットにおける最小フレーム(64バイト)を衝突検出するためのケーブル長（短縮率0.77）の最大長$L_{max}$を求めよ

        $$ケーブル上の信号の速度\upsilon_s$$
        $$=3.0\times10^8 \lbrack ^m/_s \rbrack \times 0.77 = 2.3 \times 10^8 \lbrack ^m/_s \rbrack $$

        $\frac{64\times8}{10\times10^6}=5.12\times10^{-5}s$
        $0.77\times3.0\times10^8=2.3\times10^8m/s$
        $L_{max}=\frac{1}{2} \times5.12\times10^{-5}\times2.3\times10^8=5.9\times10^3m$
    - ## 伝播遅延とケーブル長
         伝送速度を10Mbpsとし、伝送遅延μsのリピータ2台で3本のケーブルを接続するとき,イーサーネットにおける最小フレームを衝突検出するためのケーブル（短縮率0.77）の最大長$L_{max}$を求めよ
        $
        t_s \geq \frac{6L}{v_s} + 4d_r
        $

        $
        L \leq \frac{1}{6}(t_s - 4d_r)v_s
        $

        $
        t_s = \frac{512\ \mathrm{bit}}{10^7\ \mathrm{bps}}
        = 51.2 \times 10^{-6}\ \mathrm{s}
        $

        $
        v_s = 3.0 \times 10^8 \times 0.77\ \mathrm{m/s}
        = 2.3 \times 10^8\ \mathrm{m/s}
        $

        $
        d_r = 9.0 \times 10^{-6}\ \mathrm{s}
        $

        $
        L_{\max}
        = \frac{1}{6}(51.2 - 4 \times 9) \times 10^{-6}
        \times 2.3 \times 10^8
        = 580\ \mathrm{m}
        $
   - ## Ethernetのフレーム構造
        ```
                              ________________MACアドレス_____________
                             ||                                     ||
                 7       1   ||       6                   6         ||  2        0-1500              0-46      4
             _____________________________________________________________________________________________________________
            |          |     |                   |                   |     |                     |          |             |
            | Preamble |     | Destination       |  Source           |     |       D a t a       |   Pad    |  CRC        |
            |          |     | address           |  address          |     |                     |          |             |
            |__________|_____|___________________|___________________|_____|_____________________|__________|_____________|

                          ^                                             ^
                          Stat of framae                          Type/Length of data field
        ```
        - 信号のタイミングを合わせるためにPreambleをフレームの前に送信する
            7バイトの**10101010**の後に1バイトの**10101011**を送出 
        - エラー検出にはCRCが使われる
          - エラーを検出したらフレームを破棄する
        - 宛先と送信元はそれぞれ6ビットの**MACアドレス**で指定する
          - 受信したフレームの宛先が**自分のアドレス**または**ブロードキャスト**なら**上位プロトコル**へ渡す
          - そうでなければ破棄する
        - フレーム長の最大は1500バイト
          - 最小は64バイトになるようにPadが付加される(64-(6×2+2+4)=46)
        - Type部分で**上位プロトコル**が指定される
    - ## 信頼性の低いコネクションレスのサービス
      - コネクションレス
        - 送受信ノード間でハンドシェークをしない
      - 信頼性が低い
        - 受信のーどは送信のーどに確認応答も否定応答も返さない
        - 上位層に渡すパケットには一部欠損があるかもしれない
        - 欠損はトランスポート層で対処する
- # イーサネットの高速化
    当初は10Mbpsだったが必要に応じてスピードはどんどん上がっていった。そのためにどのような工夫がされたのかを見ていきましょう
    - UTPの伝送損失は周波数が高くなるほど大
    - 広く使われているケーブルの帯域は100MHz

    - 光ファイバーの使用     
    - ケーブルの改良 250MHz 
    - コストが高くつく

    - 伝送方式の改良　様々な工夫
    - イーサネット系列表示法
        ```
            例 1000 BASE - T
                |   |      |
                |   |      |-伝送媒体
                |   |       　5 同軸ケーブル(500m)
                |   |       　2 同軸ケーブル(185m)
                |   |       　T UTPケーブル
                |   |       　F 光ファイバー　などなど
                |   |- 伝送方式
                |       BASE  ベースバンド伝送
                |       BROAD ブロードバンド伝送
                |-伝送速度
                    10   10 Mbps
                    100  100Mbps
                    1000 1 Gbps
                    10G  10Gbps
        ``` 
    - ### 100Mbpsイーサネット
      - 100BASE-FX
        - NRXI:1を伝送するときに信号変化
        - 4B/5B
          - 5ビットのコード(32種類)のうち、条件を満たすものを選択
          - 4ビット(16種類)のコードに割り当てる
            - 先頭に2つ以上の0がない
            - 末尾に3つ以上の0がない
          - 選ばれなかったビットパターンを他の目的に使える
            - 11111アイドル、00000不通(断線)
            - 00100停止
        - **メディア上の伝送レートは1.25倍**=125Mbps
      - ### 100BASE-TX
        - MLT-3:3レベルの多値信号、1を伝送するときレベル変化
        - メディア上の信号帯域:**マンチェスター符号の1/4**
      - ### 1000BASE-SX,1000BASE-LX
        - 8B/10B,NRZ
        - メディア上の伝送速度:1.25Gbps
        - 波長の違い
          - 850nm: 1000BASE-SX (short)
            - マルチモード:~550m
          - 1300nm:1000BASE-LX
            - マルチモード:~550m
            - シングルモード:~5km
      - ###  1000BASE-T
        - 4D-PAM5,8B/1Q4
          1. 8ビットデータに冗長ビットを付加して9ビットにする
          2. 9ビットデータを5進数4桁に変換
              $2^9512<625=5^4$ 
          3. 5進数4桁を4組のケーブルに分配
          4. 5進数1桁を5レベルの信号で伝送
              メディア上の信号帯域: 1GHz/8/2=62.5MHz
      - ###  10Gbpsイーサネット
        - WAN PHY:既存の公衆通信ネットワーク(SONET/SDH)でフレームを伝送
        - LAN PHY:Ethernetフレームをそのまま伝送
          - #### 10GBASE-R
            - 64B/66zb:64ビットのデータにスクランブルをかけ、区切り2ビットを追加
            - 3.1125%のオーバヘッド(8B/10Bは25%)
          - #### 10GBASE-X
            - 8B/10Bを4波長で伝送
            - 12.5G/4=3.125Gbps
      - ### 40G/100Gイーサネット
        - 複数のケーブルで伝送
          - 4G:10.31G×4本
          - 100G10.31G×10本
        - 複数の波長で伝送
          - 40G:10.31G×4波長
          - 100G:25.78G×4波長
