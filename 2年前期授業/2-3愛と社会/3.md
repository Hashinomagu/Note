## 探索アルゴリズムの性能と評価尺度
- アルゴリズム
  - 完全性
    - ある問題を探索によって解を見出す
      - その問題に本当に解がある？
      - 解がある前提で解いているわけではないことが多い
        - 解が見つかった
        - 解があるけど見つからなかった
        - 解がないから見つからなかった
    - 解が存在するならは必ず見つけてくれる
    - 解が存在しないのならば必ず存在しないことがわかる
  - 最適性
    - その問題に最適解を見つけることを保証するかどうか
    - 
  - 上記は満たしているから優秀、偉いとかそういうものではない
  - アルゴリズムの良し悪しをはかる評価尺度
    - 時間計算量
      - 大きい問題であればあるほど時間は長くなるがそれに対してどのくらい時間がかかるかを測る
    - 領域計算量
      - メモリ使用量
      - メモリをバカ喰いしない方が優秀になる
## 探索の効率性
- 我々が生きている間に終わらないような問題も解けるようにしたい
- 値を持っておけば分岐の時に参考になる？
  -  状態空間グラフの全体が見えている
     -  正確な数値をつけることができる
  -  状態空間グラフの全体をわかってるということはあり得ない
     -  厳密に正しい値をつけることは現実的に不可能
        -  予測する必要がある
-  #### ヒューリスティックス関数
   -  を用いて予想をする
   -  分岐に対して厳密な値をつけることは不可能
      -  経験や傾向などの知識を活用して数値をつける必要がある 
   -  発見的探索
      -  山登り法
        山登り法には他嶺性（峰がたくさんあるようなもの）の問題は苦手、また完全性の保証もない（解があるにも関わらず見つからない恐れもある）
         -  今いる状態を展開する
         -  ヒューリスティックス関数値が一番小さいものを一つ選ぶ
         -  単純で余計なことをしていない
            -  めちゃくちゃ高速
            -  めちゃくちゃコーディングも楽
            -  データ構造的にもシンプル
         -  解を見つけたとしても必ずしも最適解かどうかはわからない

         - 手っ取り早く準最適な解を探すのに有効
         - 問題
           - 最適性は保証されていないアルゴリズム
             - 原因
               - 状態空間上のコストを考慮されていない
                 - だから単純で高速
               - ヒューリスティック関数が適切でない
                 - あくまで予測値だから
           - 局所最適解への補足
             - 一部のところで無限ループに陥ること
             - ヒューリスティック関数値をつけ間違えてしまった場合などで起こりやすい
             - 最適とついてるからいいように聞こえるけど局所最適は良くない
         - 山登り法は劣等生？
           - そうではない
             - とにかくはやい
             - 問題によっては使える
             - アルゴリズムはシンプルなので実装しやすい
        - 局所最適解を防ぎたい
          - 完全性、最適性が保証されているアルゴリズムを使う
            - 問題に対するかいや最適解を必ず保証されている
              - しかし理論的に見つけられたとしても時間が膨大にかかる
                - 解けたことを知らずにタヒんでいく
                - 下手すると何億年もかかるから
            - 分枝限定法、A*アルゴリズムなど
        - メタヒューリスティクス
          - メタ：「超~」
          - ヒューリティクスをうまく利用するためのヒューリスティクス
          - 必ず解を得られるとは限らないが、高速に順最適解の導出ができるかも
      -  確率的山登り法
         -  山登り法は絶対で上に行くが、確率的は時々下に下がる
- 遺伝的アルゴリズム（GA）
  - 生物の進化過程を模倣
  - さまざまなものの中から環境に適応したものだけが生き残る
    ```
        ________      ________
        |       |     |       |
        |       | __\ |       |
        | 誕 生　| --/ | 淘 汰　| 
        |       |     |       |
        |       | /__ |       |
        |       | \-- |       |
        |_______|     |_______|   
    ``` 
  - 問題構造に対して頑丈
    - 解き方がわからなくても大丈夫
  - 応用範囲は広い
    - 最適化、データマイニング、機械学習、画像処理
